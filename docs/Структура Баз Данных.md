1. **cities** - справочник городов  

| Поле   | Тип данных   | Описание        |     |
| ------ | ------------ | --------------- | --- |
| id     | BIGINT       | Первичный ключ  |     |
| name   | varchar(100) | название города |     |
| region | varchar(100) | регион          |     |

Создание таблицы
```sql
CREATE TABLE IF NOT EXISTS cities (
    id     BIGSERIAL PRIMARY KEY,
    name   VARCHAR(100) NOT NULL,
    region VARCHAR(100),
    UNIQUE (name, region)
);
```

Вставка
```sql
INSERT INTO cities(name, region) VALUES (?, ?);
```

Обновление
```sql
UPDATE cities SET name = ?, region = ? WHERE id = ?;
```

Удаление
```sql
DELETE FROM cities WHERE id = ?;
```

| ----------- | ------------ | ----------------------- | --- |
| id          | BIGINT       | Первичный ключ          |     |


Обновление
```sql
UPDATE firms SET name = ?, inn = ?, phone = ? WHERE ati_firm_id = ?;
```

Удаление
```sql

```

6. **loads** - таблица, которая хранит данные о грузах (для анализа)  

| Поле              | Тип данных    | Описание                                   |     |
| ----------------- | ------------- | ------------------------------------------ | --- |
| id                | BIGINT        | Первичный ключ                             |     |
| ati_load_id       | varchar(36)   | ID груза в системе ATI (для синхронизации) |     |
| cargo_type        | varchar(100)  | наименование груза                         |     |
| weight            | NUMERIC(10,2) | вес груза в тоннах                         |     |
| volume            | NUMERIC(10,2) | объём груза в м³                           |     |
| loading_city_id   | BIGINT        | Внешний ключ к таблице cities              |     |
| unloading_city_id | BIGINT        | Внешний ключ к таблице cities              |     |
| firm_id           | BIGINT        | Внешний ключ к таблице firms               |     |
| rate              | NUMERIC(12,2) | ставка за перевозку                        |     |
| status            | varchar(20)   | статус груза (new, taken, done)            |     |
| first_date        | DATE          | дата, с которой груз актуален              |     |
| last_date         | DATE          | дата, после которой груз не актуален       |     |
| synced_at         | TIMESTAMP     | время последней синхронизации с API        |     |

Создание таблицы
```sql
CREATE TABLE IF NOT EXISTS loads (
    id                BIGSERIAL PRIMARY KEY,
    ati_load_id       VARCHAR(36) UNIQUE,
    cargo_type        VARCHAR(100) NOT NULL,
    weight            NUMERIC(10,2),
    volume            NUMERIC(10,2),
    loading_city_id   BIGINT NOT NULL REFERENCES cities(id),
    unloading_city_id BIGINT NOT NULL REFERENCES cities(id),
    firm_id           BIGINT REFERENCES firms(id) ON DELETE SET NULL,
    rate              NUMERIC(12,2),
    status VARCHAR(20) NOT NULL DEFAULT 'new' CHECK (status IN ('new', 'taken', 'done')),
    first_date        DATE,
    last_date         DATE,
    synced_at         TIMESTAMP NOT NULL DEFAULT now()
);
```

Вставка (с учётом повторной синхронизации — обновит запись, если груз уже есть)
```sql
INSERT INTO loads(ati_load_id, cargo_type, weight, volume, loading_city_id, unloading_city_id, firm_id, rate, status, first_date, last_date, synced_at)
    VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, now())
ON CONFLICT (ati_load_id) DO UPDATE SET
    rate = EXCLUDED.rate,
    status = EXCLUDED.status,
    last_date = EXCLUDED.last_date,
    synced_at = now();
```

Обновление
```sql
UPDATE loads SET cargo_type = ?, weight = ?, volume = ?, loading_city_id = ?, unloading_city_id = ?,
    firm_id = ?, rate = ?, status = ?, first_date = ?, last_date = ?
    WHERE id = ?;
```

Удаление
```sql
DELETE FROM loads WHERE id = ?;
```

7. **assignments** - таблица, которая хранит данные о том, кто и на чём везёт конкретный груз

| Поле      | Тип данных | Описание                       |     |
| --------- | ---------- | ------------------------------ | --- |
| id        | BIGINT     | Первичный ключ                 |     |
| load_id   | BIGINT     | Внешний ключ к таблице loads   |     |
| driver_id | BIGINT     | Внешний ключ к таблице drivers |     |
| truck_id  | BIGINT     | Внешний ключ к таблице trucks  |     |

Создание таблицы
```sql
CREATE TABLE IF NOT EXISTS assignments (
    id         BIGSERIAL PRIMARY KEY,
    load_id    BIGINT NOT NULL REFERENCES loads(id) ON DELETE CASCADE,
    driver_id  BIGINT REFERENCES drivers(id) ON DELETE SET NULL,
    truck_id   BIGINT REFERENCES trucks(id) ON DELETE SET NULL
);
```

Вставка
```sql
INSERT INTO assignments(load_id, driver_id, truck_id)
    VALUES (?, ?, ?);

UPDATE loads SET status = 'taken' WHERE id = ?; -- обновить статус груза
```

Обновление
```sql
UPDATE assignments SET driver_id = ?, truck_id = ? WHERE id = ?;
```

Удаление
```sql
DELETE FROM assignments WHERE id = ?;
```

8. **load_boards** - на каких площадках размещён груз (многие-ко-многим)  

| Поле | Тип данных | Описание |  
| -------- | ---------- | ------------------------------- |  
| load_id | BIGINT | Внешний ключ к таблице loads |  
| board_id | BIGINT | Внешний ключ к таблице boards 

Создание таблицы
```sql
CREATE TABLE IF NOT EXISTS load_boards (
    load_id  BIGINT NOT NULL REFERENCES loads(id) ON DELETE CASCADE,
    board_id BIGINT NOT NULL REFERENCES boards(id) ON DELETE CASCADE,
    PRIMARY KEY (load_id, board_id)
);
```

Вставка
```sql
INSERT INTO load_boards(load_id, board_id) VALUES (?, ?)
ON CONFLICT DO NOTHING;
```

Удаление
```sql
DELETE FROM load_boards WHERE load_id = ? AND board_id = ?;
```

### Схема связей

| Таблица (откуда) | Поле                | Ссылается на  | Поведение при удалении родителя | Почему                                                                |
| ---------------- | ------------------- | ------------- | ------------------------------- | --------------------------------------------------------------------- |
| `drivers`        | `truck_id`          | `trucks(id)`  | `SET NULL`                      | водитель не должен исчезать из базы, если машину списали              |
| `boards`         | `owner_firm_id`     | `firms(id)`   | `RESTRICT` (по умолчанию)       | площадка без владельца не имеет смысла - сначала разберитесь с фирмой |
| `loads`          | `loading_city_id`   | `cities(id)`  | `RESTRICT` (по умолчанию)       | город - справочник, не должен удаляться, пока на него есть ссылки     |
| `loads`          | `unloading_city_id` | `cities(id)`  | `RESTRICT` (по умолчанию)       | аналогично                                                            |
| `loads`          | `firm_id`           | `firms(id)`   | `SET NULL`                      | фирма ушла с рынка - историю грузов всё равно нужно сохранить         |
| `assignments`    | `load_id`           | `loads(id)`   | `CASCADE`                       | без груза само назначение бессмысленно                                |
| `assignments`    | `driver_id`         | `drivers(id)` | `SET NULL`                      | водитель уволился - история о том, кто что вёз, должна остаться       |
| `assignments`    | `truck_id`          | `trucks(id)`  | `SET NULL`                      | машину списали - аналогично                                           |
| `load_boards`    | `load_id`           | `loads(id)`   | `CASCADE`                       | груз удалили - связи с площадками теряют смысл сами по себе           |
| `load_boards`    | `board_id`          | `boards(id)`  | `CASCADE`                       | площадку удалили (перестали с ней работать) - связи тоже не нужны     |
